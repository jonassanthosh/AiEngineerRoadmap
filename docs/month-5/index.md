# Month 5: LLM Optimization

You can train a language model. Now learn to make it smaller, faster, cheaper, and better at following instructions. This month covers the techniques used by teams shipping LLM-powered products: fine-tuning, alignment, quantization, and production deployment.

## What You'll Cover

| Lesson | Topic |
|--------|-------|
| 1 | [Fine-Tuning Strategies](fine-tuning-strategies) — full fine-tuning, LoRA, QLoRA, adapter methods |
| 2 | [RLHF](rlhf) — reinforcement learning from human feedback, DPO, preference learning |
| 3 | [Quantization](quantization) — INT8, INT4, GPTQ, AWQ, and when to use each |
| 4 | [Pruning & Distillation](pruning-distillation) — making models smaller without losing quality |
| 5 | [Inference Optimization](inference-optimization) — FlashAttention, KV caching, speculative decoding, vLLM |
| 6 | [Serving & Deployment](serving-deployment) — Ollama, vLLM, REST APIs, production monitoring |
| 7 | [**Project: Fine-Tune and Deploy an LLM**](month5-project) — end-to-end from base model to API |

## Prerequisites

Months 1–4, especially the GPT architecture, nanoGPT, and training infrastructure from Month 4.

## Capstone Project

You'll take a base model, fine-tune it with QLoRA on a custom instruction dataset, quantize the result, deploy it as a REST API, and evaluate the final model — the full workflow used by production teams.

## Estimated Time

**~60–80 hours** total (15–20 hours/week over 4 weeks).
