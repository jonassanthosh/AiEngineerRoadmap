# AI Engineering Academy Curriculum

A structured, six-month learning path that takes you from the fundamentals of AI to building, optimizing, and deploying your own large language models.

## Who This Is For

- **Career switchers** moving into AI/ML engineering from software development or another technical field
- **Students** looking for a hands-on complement to academic coursework
- **Working engineers** who want to understand LLMs deeply, not just call APIs

## Prerequisites

You'll get the most from this curriculum if you have:

- **Programming experience** — you can write functions, use loops, and work with data structures (Python preferred, but any language works)
- **High-school math** — comfort with algebra, basic statistics, and reading equations (we teach the rest)
- **A working Python environment** — Python 3.10+ with pip (we'll install libraries as we go)

No prior machine learning experience is required. Month 1 starts from scratch.

## How the Curriculum Works

Each month covers a major topic area. Lessons build on each other within a month, and each month builds on the ones before it.

| Month | Topic | What You'll Build |
|-------|-------|-------------------|
| [**Month 1**](/curriculum/month-1/what-is-ai) | AI & ML Foundations | Your first neural network from scratch |
| [**Month 2**](/curriculum/month-2/cnns) | Deep Learning | Image classifier with CNNs and transfer learning |
| [**Month 3**](/curriculum/month-3/text-preprocessing) | NLP & Transformers | A Transformer architecture from the ground up |
| [**Month 4**](/curriculum/month-4/scaling-laws) | Large Language Models | A GPT-style language model (nanoGPT) |
| [**Month 5**](/curriculum/month-5/fine-tuning-strategies) | LLM Optimization | Fine-tune, quantize, and deploy an LLM |
| [**Month 6**](/curriculum/month-6/reading-papers) | Building New Models | Design, train, evaluate, and ship a model |

### Each Lesson Includes

- **Explanations** with math rendered inline (KaTeX) and ASCII diagrams
- **Code examples** you can run locally or in a notebook
- **Exercises** at beginner, intermediate, and advanced levels with hints and solutions
- **Curated resources** — papers, videos, courses, and tools linked directly from the text

### Each Month Ends With a Capstone Project

Projects integrate everything you learned that month into a real deliverable: training a model, building a pipeline, deploying an API, or writing a technical report.

## Pacing

The curriculum is designed for roughly **15–20 hours per week** over six months. That breaks down to:

- **~5 hours/week** reading lessons and running code examples
- **~5 hours/week** working through exercises
- **~5–10 hours/week** on the capstone project (concentrated in the final week of each month)

Go faster or slower based on your schedule. The material isn't going anywhere.

## How to Use This Site

1. **Start with Month 1, Lesson 1** — [What Is Artificial Intelligence?](/curriculum/month-1/what-is-ai)
2. **Work through lessons in order** within each month — they build on each other
3. **Do the exercises** — reading alone won't build the skills; writing code will
4. **Complete the capstone project** before moving to the next month
5. **Check the [blog](/blog)** for curated video courses, research papers, and blogs to supplement your learning

:::tip[Don't Skip the Exercises]
The single biggest predictor of success is doing the exercises, not just reading the lessons. Each exercise is designed to surface a specific misunderstanding or build a specific skill. If you're short on time, skip a lesson section before skipping an exercise.
:::
